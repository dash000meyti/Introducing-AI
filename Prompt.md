Interactive AI Presentation Platform

معرفی پروژه

یک وب‌اپلیکیشن Mobile-Only برای ارائه محتوای تعاملی که تجربه‌ای مشابه TED Talk ایجاد می‌کند.

در این سیستم یک کاراکتر مجازی روی استیج حضور دارد، اسلایدها را نمایش می‌دهد، محتوا را روایت می‌کند و در آینده قادر خواهد بود بر اساس تعامل کاربر مسیر ارائه را تغییر دهد.

نسخه اولیه (MVP) بر پایه محتوای از پیش آماده‌شده اجرا می‌شود و در نسخه‌های بعدی تولید محتوا، پاسخ به سوالات و مدیریت سناریو توسط مدل‌های زبانی (LLM) انجام خواهد شد.


اهداف پروژه

اهداف MVP

* نمایش یک ارائه از پیش آماده شده
* نمایش کاراکتر سخنران با لیپ‌سینک
* پخش صوت هماهنگ با کاراکتر
* نمایش اسلایدها
* کنترل صحنه و نورپردازی و بزرگنمایی
* نمایش متن گفتار
* ناوبری و کنترل جریان ارائه
* تغییر جریان ارائه بر اساس سوال های از پیش تایین شده

اهداف آینده

* تولید خودکار اسلایدها
* تولید خودکار متن سخنرانی
* پاسخگویی به سوالات کاربر
* تغییر مسیر ارائه بر اساس تعامل کاربر
* شخصی‌سازی محتوا بر اساس لینک ورودی یا پروفایل مخاطب


ساختار صحنه

صحنه از چند لایه مستقل تشکیل می‌شود:

A. Introducing Layer
که شامل ۴ لایه داخلش میشه

1. Backstage Layer
پایین‌ترین لایه سیستم.

وظایف:
* نمایش پس‌زمینه استیج
* پشتیبانی از تم روشن و تاریک ( ۲ تصویر با تم سیاه و سفید )
* پشتیبانی از نور پردازی ( به ازای هر تم ۴ تصویر : همه جا خاموش - همه جا روشن - استیج روشن - پرزنتیشن روشن )
* قابلیت تغییر ظاهر در آینده

2. Presentation Layer
لایه نمایش اسلایدها.

وظایف:
* نمایش اسلایدها
* پشتیبانی از تم روشن و تاریک
* پشتیبانی از نور پردازی ( دارای ۲ حالت خاموش و روشن : خاموش یعنی اوپاسیتی ۱۰ ٪ - روشن یعنی اوپاسیتی ۱۰۰ ٪ )
* نسبت نمایش محتوا ۲:۳ عمودی است
* قابلیت داشتند تمپلیت و نمایش محتوا در یکی از قابل ها در آینده

تکنولوژی:
* Reveal.js

3. Character Layer
شخصیت مجازی ارائه‌دهنده.

وظایف:
* نمایش کاراکتر
* پشتیبانی از نور پردازی ( دارای ۲ حالت خاموش و روشن : خاموش یعنی کاراکتر با رنگ بکگراند تم ( سیاه یا سفید ) - روشن یعنی نمایش معمولی )
* اجرای انیمیشن‌
    * بدن : راه رفتن و حرکت دست ها
    * سر : لیپ سینک و نمایش احساسات

تکنولوژی:
* PixiJS
* Sprite Sheets

4. Overlay Layer
المان‌های کمکی روی صحنه.

نمونه‌ها:
* Popup
* Tooltip
* Highlight
* Callout

B. Navigation Layer
پلیر کنترل ارائه.

وظایف:
* مدیریت بخش های ارائه ( بخش های مشخص با سوالات و مسیر های از پیش تعریف شده در آینده کل مسیر و تعداد بخش ها با هوش مصنوعی ساخته میشه )
* مدیریت صبر کردن بین هر بخش ( اگه قراره جریان ارائه بر اساس سوال تغییر کنه و گرنه به مسیر پیشفرض ادامه میده )
* نمایش پیشرفت ارائه ( تعداد بخش ها و پیشرفت هر بخش برای مثل تایم لاین ویدیو )
* نمایش بخش جاری ( نمایش عنوان بخش جاری ارائه )
* مدیریت صدا ( صدای کاراکتر که فعلا آماده است و در آینده با هوش مصنوعی ساخته میشه - صدایه محیطی برای بخش های خاص ارائه مثل دست زدن، خندیدن و…  )
* مدیریت موت صدا ( لیپ سینک و Transcript به کار خود ادامه میدن فقط صدا پخش نمیشود )
* مدیریت تم رنگی روشن یا تاریک
* کنترل جریان ارائه
    * مدیریت نورپردازی ( در سه لایه Backstage، Presentation و Character )
    * مدیریت اجرای انیمیشن های کاراکتر
    * مدیریت  لیپ سینک دقیق صدا و متن در Transcript
    * مدیریت محتوای داخل اسلاید
    * مدیریت روم نمایش در لایه Introducing ( ۲ مقدار بزرگنمایی مثل زوم این و زوم اوت مشخص و جابه جایی بین این دو حالت : نمایش نزدیک برای تمرکز بر صحبت های کاراکتر - نمایش دور برای تمرکز بر اسلاید )

موقعیت:
* پایین صفحه

C. Transcript Layer
نمایش متن سخنان کاراکتر.

وظایف:
* نمایش زنده متن گفتار
* کمک به کاربرانی که امکان شنیدن صدا ندارند
* نمایش جزئیات که در متن گقت و گو نیست و توسیف فضا میکنه مثل «صدای دست زدن» یا …

موقعیت:
* بالای صفحه
